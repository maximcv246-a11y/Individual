## 1. Описание проекта

### Цель

Разработать интерактивное одностраничное веб-приложение (SPA) — цифровое меню ресторана — без использования сторонних фреймворков, применив модульный подход, ES6+ синтаксис, динамическую работу с DOM и современные возможности CSS.

### Основные функции

| Функция | Описание |
|---------|----------|
| **Вкладки категорий** | Горячее 🔥 / Салаты 🥗 / Десерты 🍮 — переключение без перезагрузки страницы |
| **Поиск** | Фильтрация блюд по названию и описанию в реальном времени (debounce 300 мс) |
| **Фильтрация** | По тегам: Все / Вегетарианское / Острое / Популярное |
| **Сортировка** | По цене (возр./убыв.), по названию, по калориям |
| **Корзина заказов** | Добавление с выбором количества, удаление позиций, подсчёт итоговой суммы |
| **Детальный просмотр** | Модальное окно с полным описанием блюда |
| **Уведомления** | Toast-сообщения о добавлении в корзину и оформлении заказа |
| **Анимации** | Staggered-появление карточек, hover-эффекты, bump-анимация счётчика |
| **Адаптивность** | Корректное отображение на мобильных устройствах (breakpoint 680px) |

### Технологический стек

- **HTML5** — семантическая разметка, ARIA-атрибуты
- **CSS3** — Custom Properties, Grid, Flexbox, Keyframe-анимации, backdrop-filter
- **JavaScript ES6+** — модули, Map, arrow functions, destructuring, template literals, spread

## 3. Документация к проекту

### 3.1 Архитектура модулей

```
app.js (точка входа)
  ├── импортирует → data.js     (данные)
  ├── импортирует → ui.js       (рендер)
  │     └── импортирует → cart.js  (корзина)
  └── управляет состоянием: activeTab, activeFilter, searchQuery, sortOrder
```

### 3.2 Модуль `data.js`

Содержит массив из 18 объектов-блюд. Каждый объект имеет следующую структуру:

```js
{
  id: 1,                        // уникальный идентификатор
  category: 'hot',              // 'hot' | 'salads' | 'desserts'
  name: 'Утиная грудка',        // название блюда
  desc: 'Карамелизованная...',  // описание
  price: 540,                   // цена в рублях
  calories: 620,                // калорийность (ккал)
  weight: 280,                  // вес порции (г)
  emoji: '🍗',                  // иконка блюда
  bgColor: '#fef3e2',           // цвет фона карточки
  tags: ['popular']             // теги: 'popular' | 'vegetarian' | 'spicy'
}
```

**Распределение блюд по категориям:**

| Категория | Количество | ID блюд |
|-----------|-----------|---------|
| Горячее (`hot`) | 6 | 1–6 |
| Салаты (`salads`) | 6 | 7–12 |
| Десерты (`desserts`) | 6 | 13–18 |

### 3.3 Модуль `cart.js`

Управляет состоянием корзины через `Map<id, {item, qty}>`.

| Экспортируемая функция | Параметры | Возвращает | Описание |
|------------------------|-----------|-----------|----------|
| `addToCart(item, qty)` | `item: Object`, `qty: number = 1` | `void` | Добавить блюдо или увеличить qty |
| `removeFromCart(id)` | `id: number` | `void` | Удалить позицию полностью |
| `clearCart()` | — | `void` | Очистить корзину |
| `getCartItems()` | — | `Array` | Получить все позиции |
| `getCartCount()` | — | `number` | Сумма всех qty |
| `getCartTotal()` | — | `number` | Итоговая сумма в рублях |

### 3.4 Модуль `ui.js`

Отвечает за все DOM-операции.

| Экспортируемая функция | Описание |
|------------------------|----------|
| `renderMenu(items)` | Отрисовать массив блюд в `.menu-grid` |
| `bindCardEvents(dishMap)` | Навесить делегированные обработчики на сетку |
| `openDetailModal(dish)` | Открыть детальный просмотр блюда |
| `refreshCartCount()` | Обновить счётчик без анимации |
| `showToast(message, duration)` | Показать уведомление |

**Принцип делегирования событий:**

```js
// Один обработчик на весь grid вместо N обработчиков на каждой карточке
menuGrid.addEventListener('click', (e) => {
  const id = Number(e.target.dataset.id);
  // определяем тип действия по dataset.action или classList
```

### 3.5 Модуль `app.js`

Главный модуль. Хранит состояние приложения:

```js
const state = {
  activeTab:    'hot',      // текущая вкладка
  activeFilter: 'all',      // текущий фильтр
```

Pipeline обработки данных:


## 4. Примеры использования

### 4.1 Переключение вкладок

Нажатие на вкладку меняет `state.activeTab` и перерисовывает сетку:

```js
tabBtns.forEach(btn => {
  btn.addEventListener('click', () => {
    tabBtns.forEach(b => {
      b.classList.remove('active');
      b.setAttribute('aria-selected', 'false');
    });
    btn.classList.add('active');
    btn.setAttribute('aria-selected', 'true');

    state.activeTab = btn.dataset.tab;
    // Сброс фильтра при смене вкладки
    state.activeFilter = 'all';
    filterBtns.forEach(f =>
      f.classList.toggle('active', f.dataset.filter === 'all')
    );

    applyAndRender();
  });
});
```

**Результат:** при клике на «Салаты» отображаются только 6 блюд категории `salads`, фильтр сбрасывается на «Все».

---

### 4.2 Поиск с debounce

```js
let searchDebounce = null;

searchInput.addEventListener('input', () => {
  clearTimeout(searchDebounce);
  searchDebounce = setTimeout(() => {
    state.searchQuery = searchInput.value;
    applyAndRender();
  }, 300); // ждём 300 мс после последнего нажатия
});
```

**Результат:** при вводе «тир» в строку поиска отобразится только «Тирамису». Лишние запросы при быстром наборе не выполняются.

### 4.3 Добавление блюда в корзину

```js
// cart.js
export function addToCart(item, qty = 1) {
  if (cartMap.has(item.id)) {
    // Если блюдо уже в корзине — увеличиваем qty
    cartMap.get(item.id).qty += qty;
  } else {
    cartMap.set(item.id, { item, qty });
  }
}
```

**Пользовательский сценарий:**
1. Пользователь устанавливает количество `2` с помощью кнопок `+`/`−`
2. Нажимает «В корзину»
3. Счётчик на кнопке корзины увеличивается на `2` с анимацией
4. Появляется toast: *«Утиная грудка добавлено в корзину!»*

---

### 4.4 Фильтрация и сортировка (полный pipeline)

```js
function applyAndRender() {
  // Шаг 1: фильтр по категории (вкладка)
  let result = menuItems.filter(d => d.category === state.activeTab);

  // Шаг 2: фильтр по тегу
  if (state.activeFilter !== 'all') {
    result = result.filter(d => d.tags.includes(state.activeFilter));
  }

  // Шаг 3: поиск по тексту
  const q = state.searchQuery.trim().toLowerCase();
  if (q) {
    result = result.filter(d =>
      d.name.toLowerCase().includes(q) ||
      d.desc.toLowerCase().includes(q)
    );
  }

  // Шаг 4: сортировка
  result = sortItems(result, state.sortOrder);

  // Шаг 5: рендер или заглушка «не найдено»
  renderMenu(result);
}
```

## 5. Ответы на контрольные вопросы

**1. Что такое ES-модули и зачем они используются в проекте?**

ES-модули (`import`/`export`) — стандартный механизм JavaScript для разделения кода на независимые файлы. В проекте используются для изоляции ответственности: `data.js` хранит данные, `cart.js` — логику корзины, `ui.js` — DOM-операции, `app.js` — координацию. Это упрощает сопровождение и тестирование кода.

**2. Почему нельзя открыть `index.html` напрямую через `file://`?**

Браузеры блокируют cross-origin запросы для протокола `file://` из соображений безопасности. ES-модули при загрузке через `file://` получают origin `null`, что вызывает CORS-ошибку. Решение — запускать через HTTP-сервер.

**3. Что такое делегирование событий и где оно применяется в проекте?**

Делегирование событий — приём, при котором один обработчик вешается на родительский элемент вместо множества обработчиков на дочерние. В проекте `ui.js` использует один `click`-обработчик на `.menu-grid` для всех кнопок «В корзину», `+`/`−` и кликов по карточкам. Это снижает потребление памяти и работает с динамически добавленными элементами.

**4. Что такое debounce и зачем он нужен в поиске?**

Debounce — техника задержки выполнения функции до момента, когда пользователь прекратил ввод. Без него `applyAndRender()` вызывалась бы при каждом нажатии клавиши (10–20 раз при быстром наборе), что вызывало бы лишние перерисовки DOM. С debounce 300 мс функция вызывается один раз после окончания ввода.

**5. Почему для корзины используется `Map`, а не обычный массив?**

`Map` обеспечивает O(1) доступ по ключу (id блюда) при добавлении и поиске, тогда как поиск в массиве — O(n). Кроме того, `Map` автоматически предотвращает дублирование: `cartMap.has(id)` проверяет наличие блюда и обновляет qty вместо создания новой записи.

**6. Как реализована адаптивность и что такое mobile-first?**

Адаптивность реализована через CSS Grid с `auto-fill` и `minmax(280px, 1fr)`, что автоматически перестраивает колонки. Дополнительно media query `@media (max-width: 680px)` уменьшает сетку до одной колонки, скрывает лишние элементы и укрупняет кнопки для touch-управления. Mobile-first означает, что базовые стили ориентированы на маленькие экраны, а расширения — для больших.

**7. Что такое ARIA-атрибуты и зачем они добавлены?**

ARIA (Accessible Rich Internet Applications) — набор HTML-атрибутов для вспомогательных технологий (скринридеров). В проекте используются: `role="tablist"` / `role="tab"` для вкладок, `aria-selected` для состояния вкладки, `aria-label` для кнопок без текста (корзина, закрытие модала), `aria-modal="true"` для модальных окон. Это обеспечивает доступность для людей с ограниченными возможностями.

### Решения, принятые в ходе разработки

**Почему Map для корзины, а не массив?**
Map гарантирует уникальность ключей (id блюд), обеспечивает O(1) доступ и удобно обновляет существующие записи без поиска по индексу.

**Почему делегирование событий в сетке?**
Карточки перерисовываются при каждом изменении (поиск, смена вкладки). Если вешать обработчик на каждую кнопку, они теряются при `innerHTML = ''`. Делегирование на постоянный родительский элемент решает эту проблему.

**Почему debounce именно 300 мс?**
Это стандартное значение для поиска: достаточно быстро, чтобы пользователь не ждал реакции, и достаточно долго, чтобы избежать лишних вызовов при наборе.
